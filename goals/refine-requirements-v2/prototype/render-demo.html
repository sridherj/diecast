The job

Turn requirements refinement into a workflow-aware, HTML-first, living system — graspable in two minutes and iterated through inline comments, not markdown edits.

Outcomes
  • The refined output leads with the WHAT; implementation ideas are confined to a non-binding "Directional" section.
  • Each goal is classified into a work family that shapes the document and routes it into the right downstream workflow.
  • Requirements render as a designed HTML document — L1/L2/L3 hierarchy, progressive disclosure, a classification pill.
  • Inline comments drive new versions, and requirements stay the living source of truth via downstream round-trip write-back.

✓ In scope

  • Classifier agent + HTML render + comment/version engine
  • Phase-agnostic router seam + named stubs
  • Round-trip write-back receiver (simulated emitter)

✕ Out of scope

  • Building the family-specific downstream pipelines
  • Multi-user realtime collaborative editing
  • Rewriting downstream agents (planner, task-suggester)
