# Sub-phase 3: Alembic migrations + boot-time `_ensure_db_at_head()`

> **Pre-requisite:** Read `docs/execution/cast-server-first-run-launch/_shared_context.md` before starting.

## Objective

Introduce Alembic to cast-server with a hand-authored baseline migration that `op.execute()`s the existing `schema.sql` (Decision #14 — schema-first codebase, no ORM models to autogenerate from). Add a boot-time skip-if-stamped check (Decision #15) so most cast-server starts incur a single cheap `SELECT alembic_version` and only actual schema drift triggers the lock. Add migration tests that catch silent schema drift for the life of the project.

## Dependencies

- **Requires completed:** None.
- **Assumed codebase state:** `cast-server/cast_server/db/schema.sql` is present (~106 lines, 6 tables, 14 indexes). DB lives at `~/.cast/diecast.db`. No `alembic/` tree exists yet. `bin/run-migrations.py` exists but is being obsoleted.

## Scope

**In scope:**
- Add `alembic` to cast-server's runtime dependencies.
- Generate `cast-server/alembic/` and `cast-server/alembic.ini` via `uv run alembic init alembic`.
- Hand-author baseline migration that reads `schema.sql` and `op.execute()`s each statement.
- Add `_ensure_db_at_head()` to `cast-server/cast_server/main.py`; call before FastAPI accepts requests.
- One-time stamp logic for pre-Alembic existing DBs.
- `tests/test_migrations.py` — round-trip + idempotency.

**Out of scope (do NOT do these):**
- Editing `schema.sql` itself. It remains the source of truth.
- Running `alembic upgrade head` from `setup` — sp8 owns that invocation.
- Removing `bin/run-migrations.py` — sp6 marks it deprecated in its docstring; full removal is out-of-plan.
- Introducing SQLAlchemy ORM models (explicit out-of-scope per the plan).

## Detailed Steps

### Step 3.1: Add `alembic` dependency

In `cast-server/pyproject.toml`, add `alembic` (latest stable, e.g. `>=1.13`) to `[project.dependencies]`. Run `uv sync` and commit the resulting `uv.lock` change.

### Step 3.2: Initialize Alembic tree

```bash
cd cast-server
uv run alembic init alembic
```

This creates `alembic/`, `alembic.ini`, `alembic/env.py`, `alembic/script.py.mako`, `alembic/versions/`.

### Step 3.3: Configure `alembic.ini`

Set `sqlalchemy.url` to read from `cast_server.config` so it always uses the same DB the runtime uses. The cleanest pattern: leave a placeholder in `alembic.ini` and set the URL at runtime in `env.py`:

```python
# alembic/env.py top of run_migrations_*
from cast_server.config import DB_URL  # or DATABASE_URL — match what config.py exports
config.set_main_option("sqlalchemy.url", DB_URL)
```

If `config.py` doesn't yet export the URL, add an `os.fspath`-compatible accessor (e.g. `DB_URL = f"sqlite:///{Path.home() / '.cast' / 'diecast.db'}"`). Match whatever `config.py` already does for the runtime path — DRY trumps duplication.

### Step 3.4: Hand-author the baseline migration

Create `cast-server/alembic/versions/<auto-generated-rev>_baseline.py` (the `revision` value will be assigned by Alembic on `revision --autogenerate` — but we are NOT using autogenerate; just create the file manually with a fresh hex revision and `down_revision = None`).

```python
"""baseline: initial schema from schema.sql"""
from pathlib import Path
from alembic import op

revision = "<12-hex-chars>"
down_revision = None
branch_labels = None
depends_on = None

def upgrade():
    schema_sql = (
        Path(__file__).resolve().parent.parent.parent
        / "cast_server" / "db" / "schema.sql"
    )
    raw = schema_sql.read_text()
    # Naive split is correct for schema.sql (no semicolons inside string literals).
    for stmt in raw.split(";"):
        stmt = stmt.strip()
        if stmt:
            op.execute(stmt)

def downgrade():
    raise NotImplementedError(
        "baseline cannot be downgraded; rm ~/.cast/diecast.db to reset"
    )
```

**Verify** before committing:

```bash
cd cast-server
uv run alembic upgrade head    # should run schema.sql, leave alembic_version populated
sqlite3 ~/.cast/diecast.db ".schema" > /tmp/alembic_built.sql
sqlite3 :memory: < cast-server/cast_server/db/schema.sql && \
  sqlite3 :memory: ".schema" > /tmp/raw_schema.sql
diff /tmp/alembic_built.sql /tmp/raw_schema.sql   # expect no semantic differences
```

If `schema.sql` contains a triggers / partial indexes / multi-statement constructs that the naive `split(";")` mishandles, switch to a regex-aware splitter or invoke sqlite directly via `op.get_bind().exec_driver_sql(raw)` once.

### Step 3.5: Add `_ensure_db_at_head()` to `cast-server/cast_server/main.py`

```python
from pathlib import Path
from alembic import command
from alembic.config import Config
from alembic.script import ScriptDirectory
from alembic.runtime.migration import MigrationContext
from sqlalchemy import create_engine

from cast_server.config import DB_URL, CAST_SERVER_ROOT  # or wherever the package root is

ALEMBIC_INI = CAST_SERVER_ROOT / "alembic.ini"

def _ensure_db_at_head() -> None:
    cfg = Config(str(ALEMBIC_INI))
    cfg.set_main_option("sqlalchemy.url", DB_URL)
    script = ScriptDirectory.from_config(cfg)
    head = script.get_current_head()
    engine = create_engine(DB_URL)
    with engine.connect() as conn:
        ctx = MigrationContext.configure(conn)
        current = ctx.get_current_revision()
    if current is None:
        # Pre-Alembic DB (existing user with manually-created schema). Stamp baseline.
        try:
            command.stamp(cfg, "head")
            return
        except Exception:
            pass  # Fall through to upgrade if stamp fails.
    if current != head:
        command.upgrade(cfg, "head")
```

Call `_ensure_db_at_head()` exactly once at app startup, **before** the FastAPI app starts accepting requests. The cleanest hook point depends on `main.py`'s structure — likely at module top after the imports, or inside an `@app.on_event("startup")` handler. Match the existing init pattern.

### Step 3.6: One-time stamp for pre-Alembic DBs

Decision in the plan §E item 5: "the first release with Alembic also stamps existing DBs at the baseline revision via a one-time `alembic stamp head` if the `alembic_version` table is missing — wrapped in a try/except so missing-DB or empty-DB cases also work." The `current is None` branch in step 3.5 covers the "table missing or empty" case.

For a fully-empty DB (file doesn't exist or has zero tables), the stamp will fail because `alembic_version` cannot be created without the schema being present. Order the logic so:

1. If file doesn't exist → `command.upgrade(cfg, "head")` creates everything.
2. If file exists with `alembic_version` table → skip-if-stamped (the common case).
3. If file exists *without* `alembic_version` but *with* the cast tables → `command.stamp(cfg, "head")` claims the existing schema as baseline-equivalent.

A simple heuristic: try `command.upgrade` first; if it fails because tables already exist, try `command.stamp head`; if THAT fails, log loudly and re-raise. The exact branch logic should be tested in `test_migrations.py` step 3.7.

### Step 3.7: `tests/test_migrations.py` (Decision #16)

Two cases:

```python
"""Migration round-trip + idempotency tests."""
import sqlite3
import subprocess
from pathlib import Path

import pytest

ALEMBIC_INI = Path(__file__).parent.parent / "cast-server" / "alembic.ini"
SCHEMA_SQL = Path(__file__).parent.parent / "cast-server" / "cast_server" / "db" / "schema.sql"

def _alembic(tmp_db, *args):
    env_url = f"sqlite:///{tmp_db}"
    return subprocess.run(
        ["uv", "run", "alembic", "-c", str(ALEMBIC_INI), *args],
        cwd=ALEMBIC_INI.parent,
        env={**os.environ, "DIECAST_DB_URL": env_url},  # or whatever override env-var
        check=True,
        capture_output=True,
        text=True,
    )

def _schema_dump(db_path):
    out = subprocess.check_output(["sqlite3", str(db_path), ".schema"], text=True)
    # Strip alembic_version table from comparison
    return "\n".join(l for l in out.splitlines() if "alembic_version" not in l)

def test_round_trip(tmp_path):
    """alembic upgrade head from empty DB → schema matches schema.sql byte-for-byte."""
    alembic_db = tmp_path / "alembic.db"
    raw_db = tmp_path / "raw.db"
    _alembic(alembic_db, "upgrade", "head")
    conn = sqlite3.connect(raw_db)
    conn.executescript(SCHEMA_SQL.read_text())
    conn.close()
    assert _schema_dump(alembic_db) == _schema_dump(raw_db)

def test_idempotent(tmp_path):
    """Running alembic upgrade head twice is a no-op."""
    db = tmp_path / "x.db"
    _alembic(db, "upgrade", "head")
    before = _schema_dump(db)
    _alembic(db, "upgrade", "head")
    after = _schema_dump(db)
    assert before == after
```

Adjust the env-var override mechanism to whatever `cast_server.config` actually exposes for DB-URL injection — the test must avoid touching `~/.cast/diecast.db`.

If config doesn't currently support env override, add a single line: `DB_URL = os.environ.get("DIECAST_DB_URL", default_url)`.

## Execution Notes

- The `split(";")` approach in `upgrade()` is correct for `schema.sql` *only because* it has no string literals containing semicolons. If `schema.sql` later gains a CHECK constraint with `';'` in a string default, the splitter must be replaced. Add a comment in the migration noting this assumption.
- `schema.sql` is the source of truth; the migration READS it. Future schema changes are hand-authored migrations (or autogen if ORM models are introduced — out of scope here).
- Be careful with the boot-time check's race window. Two concurrent cast-server starts: the first acquires the SQLite write lock during `command.upgrade`; the second sees `current == head` once the first commits. Document this in a comment near `_ensure_db_at_head()`.
- Don't remove `bin/run-migrations.py` — sp6 will mark it deprecated in its docstring; full removal is a follow-up.
